# Asset restructure, descriptive rename, and EXIF stripping — Design

- **Date:** 2026-06-19
- **Status:** proposed
- **Repo:** oz-blog (`posts.oztamir.com`, static Astro)

## Goal

Two related improvements to the blog's media assets:

1. **Reorganize + rename** every post asset so it lives in a per-post folder with a
   short, descriptive filename — replacing the date folders and generic names
   (`Screen-Shot-…`, `IMG_…`, `DraggedImage…`, `ezgif…`, `DALL-E…`, `unnamed…`).
2. **Strip EXIF/metadata** so no location data (or other metadata) ships in the build,
   closing a confirmed GPS leak.

## Confirmed problem (EXIF)

`src/assets/content/images/2021/05/IMG_1373.png` (feature image of the iOS-NFC post)
contains GPS metadata. The build emits the **original** PNG to `dist/_astro/IMG_1373.*.png`
(originals are emitted alongside the sharp-generated WebP variants), and that original
retains all 26 GPS fields. The WebP variants are already stripped by sharp; originals and
passthrough files are not.

## Decisions (agreed with user)

- **Scope:** all ~258 non-descriptively-named images + all 8 videos. The ~140 already-fine
  names are left alone.
- **Naming:** short, descriptive, kebab-case, lowercase. "Close enough" is acceptable
  (speed/cost over precision). Generated by **Haiku** subagents.
- **Hierarchy:** `src/assets/content/images/posts/<slug>/<name>.<ext>`; feature image →
  `…/posts/<slug>/featured.<ext>`. Videos → `src/assets/content/media/posts/<slug>/…`.
  `<slug>` = the MDX filename without extension (already the post's URL slug).
- **EXIF:** build-time strip of emitted `dist` raster images **and** a one-time scrub of the
  committed source images. (Videos test clean and are not scrubbed.)
- **Cadence:** do the first post end-to-end, **checkpoint** with the user on naming style,
  then continue autonomously.

## Why per-post hierarchy

- Co-locates a post with its assets; the folder *is* the namespace, so filenames need no
  post-prefix, stay short, and can never collide across posts.
- **No resolver changes.** `mdx/images.ts` and `mdx/media.ts` glob `**` and resolve by
  relative path; `featureImage` resolves via `resolveImage` (`utils/images.ts`). All accept
  the new paths. Both the `posts/<slug>/…` tree-relative and `/content/images/posts/<slug>/…`
  legacy forms normalize correctly today.

## Target structure

Before → After (illustrative):

```
src/assets/content/images/2022/06/ezgif-5-05499f9ec4.gif   (already mp4 on deploy branch)
  → src/assets/content/media/posts/lumos-…/lamp-dimming-demo.mp4

frontmatter: featureImage: /content/images/2022/06/Screen-Shot-….png
  → featureImage: /content/images/posts/lumos-…/featured.png

<Figure src="2022/06/Screen-Shot-….png" />
  → <Figure src="posts/lumos-…/esphome-config.png" />
```

## Approach

### 1. Build the asset → post map

Parse every post MDX for asset references:
- `featureImage` (frontmatter, `/content/images/…` form).
- `<Figure src>`, `<Video src>`, `<Video poster>` (tree-relative `YYYY/MM/…`; counts today:
  341 Figure/Video `src`, 8 `poster`, 2 remote URLs which are skipped).

Produce `asset → [owning post slugs]`, then classify:
- **Orphans** — assets referenced by no post. **Listed and confirmed with the user before
  deletion** (not deleted silently).
- **Shared** — referenced by 2+ posts. Placed in `posts/_shared/` and referenced from each.

### 2. Naming (Haiku subagents)

Per post, dispatch a subagent with `model: haiku`, given: the post title/excerpt and, for
each image, its current path + alt text + a few lines of surrounding MDX. The agent reads
each image and returns `{ oldRelPath: "short-kebab-name" }`.

Orchestrator (this session) validates every returned name: enforce kebab-case + lowercase,
ensure uniqueness within the post folder (suffix `-2`, `-3` on collision), cap length, and
**force the feature image to `featured`**. Videos are named the same way.

### 3. Move + rewrite references

- `git mv` each asset to its new per-post path with the new name (feature image →
  `featured.<ext>`); video posters co-locate under the video's `media/posts/<slug>/` folder.
- Rewrite references:
  - `<Figure>/<Video>/poster` → tree-relative `posts/<slug>/<name>.<ext>`.
  - `featureImage` → `/content/images/posts/<slug>/featured.<ext>`.

### 4. EXIF stripping

- **Source scrub (one-time, repeatable):** `exiftool -all= -overwrite_original -r
  src/assets/content/images` (install exiftool locally; lossless — strips metadata, leaves
  pixels untouched). After this, emitted originals are clean.
- **Build-time guarantee:** a new Astro integration `src/integrations/strip-image-metadata.mjs`
  on the `astro:build:done` hook. It walks `dist/_astro/**/*.{png,jpg,jpeg,webp}`; for each
  file that still carries metadata, it re-saves via **sharp** with metadata stripped (files
  already clean are skipped to keep builds fast). GIFs are skipped — the format carries no
  EXIF/GPS. Registered in `astro.config.mjs`; `sharp` added to `devDependencies` so it
  resolves on Cloudflare's builder.

### 5. Cleanup

- Update stale comments in `content.config.ts` and `mdx/images.ts` ("served verbatim from
  /public — no build-time transforms" is no longer true).
- Remove the now-empty `images/YYYY/MM` and `media/YYYY/MM` directories.

## Verification

- `npm run build` — throws on any unresolved asset reference (the primary safety net).
- `npm run check` — keep at 0 errors.
- `npm run check:links` — no broken internal links.
- **EXIF assertion** — scan `dist/_astro/**` rasters and assert **0** GPS/EXIF fields.
- Asset-size invariant still holds (no emitted file > 25 MiB).
- Spot-render the checkpoint post with `npm run preview`.

## Risks & mitigations

- **Broken references** → build + `check:links` fail loudly; moves are mechanical; verify
  after each batch.
- **Haiku name quality** → acceptable per user ("close enough"); orchestrator validates
  format/uniqueness; the batch-1 checkpoint catches style problems before the bulk runs.
- **Shared / orphan assets** → detected and surfaced, never guessed at.
- **Build-time cost of the metadata pass** → skip already-clean files; after the source
  scrub, ≈0 files need re-encoding.
- **sharp availability on Cloudflare** → added as an explicit devDependency.

## Out of scope

- Domain migration (separate session).
- Reducing redundant original-asset emission in `dist` (separate optimization).
- Video re-encoding/optimization.

## Sequencing

1. **Phase A — Restructure + rename:** asset→post map → Haiku names → `git mv` → rewrite
   references → verify. **Checkpoint after the first post.**
2. **Phase B — EXIF:** source scrub + build integration + verify.

(Phases are largely independent; A is the larger effort.)
